What is a CSV File?

CSV stands for Comma-Separated Values. It's one of the simplest and most universal ways to store tabular data.

hostname ip_address device_type location
switch-01 10.0.1.1 switch Building A
router-core 10.0.0.1 router Data Center

Python's csv module is part of the standard library - no installation needed!

The csv module provides functionality to read from and write to CSV files, handling all the parsing details for you.

Your First CSV Read

Here's the basic pattern Alex uses to read a CSV file:

read_devices.py 
import csv

# Open the CSV file
with open('devices.csv', 'r') as file:
    # Create a CSV reader object
    reader = csv.reader(file)
    
    # Skip the header row
    header = next(reader)
    print(f"Columns: {header}")
    
    # Read each row
    for row in reader:
        hostname = row[0]
        ip_address = row[1]
        device_type = row[2]
        location = row[3]
        
        print(f"Found device: {hostname} at {ip_address}")

üîç Code Breakdown:

  • import csv - Import the built-in CSV module
  • open('devices.csv', 'r') - Open file in read mode
  • csv.reader(file) - Create a reader that parses the file
  • next(reader) - Get the first row (header)
  • for row in reader - Iterate through data rows

üìñ Even Better: DictReader

Accessing columns by index (row[0]) works, but it's error-prone. What if someone rearranges the columns? DictReader solves this!

read_devices_better.py 
import csv

with open('devices.csv', 'r') as file:
    # DictReader uses the header row as dictionary keys
    reader = csv.DictReader(file)
    
    for row in reader:
        # Access columns by name - much clearer!
        hostname = row['hostname']
        ip_address = row['ip_address']
        device_type = row['device_type']
        location = row['location']
        
        print(f"Device: {hostname}")
        print(f"  IP: {ip_address}")
        print(f"  Type: {device_type}")
        print(f"  Location: {location}")
        print("-" * 30)
üí° Pro Tip: Always prefer DictReader over reader when your CSV has headers. It makes your code more readable and resilient to column order changes!
