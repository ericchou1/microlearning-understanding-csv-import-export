// SYSTEM ALERT

Running your import twice causes a UNIQUE CONSTRAINT FAILURE. Devices already exist! You need idempotent operations.

THE PROBLEM
ERROR_LOG 
ERROR | UNIQUE constraint failed: dcim_device.name
Device 'tn-sw-01' already exists in database!
SOLUTION: get_or_create

Use get_or_create() ‚Äî it retrieves existing objects or creates new ones:

Idempotent Import 
# Returns (object, was_created_boolean)
device, created = Device.objects.get_or_create(
    name=hostname,  # Lookup field
    defaults={      # Fields set only when creating
        "device_type": device_type,
        "role": role,
        "location": location,
        "status": active_status,
    }
)

if created:
    self.logger.info(f"‚úì Created: {hostname}")
else:
    self.logger.info(f"‚è≠ Skipped (exists): {hostname}")
ALTERNATIVE: update_or_create

If CSV should overwrite existing data, use update_or_create():

Update Mode 
device, created = Device.objects.update_or_create(
    name=hostname,  # Lookup field
    defaults={      # Fields to set (create OR update)
        "device_type": device_type,
        "role": role,
        "location": location,
        "status": active_status,
    }
)

if created:
    self.logger.info(f"‚úì Created: {hostname}")
else:
    self.logger.info(f"üîÑ Updated: {hostname}")
COMPARISON
